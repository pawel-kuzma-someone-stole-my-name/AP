• rel – nazwa zasobu/relacji
  • href – URI do zasobu
  • method – metoda HTTP jakiej klient powinien użyć aby przejść do tej relacji 
{
     data: {...},
     links: [ { rel: "next", method: "GET", href: "http://api.ap" } ]
   }

Negocjacje za pomocą nagłówków

  • Accept - Nagłówek w żądaniu HTTP wskazuje, jakie typy zawartości, wyrażone jako typy MIME, klient jest w stanie zrozumieć. Accept: text/html,application/json,application/xml;q=0.9,*/*;q=0.8
  • Content-Type (w odpowiedzi) Nagłówek w odpowiedzi HTTP wskazuje, jaki typ zawartości, wyrażony jako typ MIME, serwer jest w stanie zwrócić.
  • Content-Type (w żądaniu) Nagłówek w żądaniu HTTP wskazuje, jaki typ zawartości, wyrażony jako typ MIME, klient wysyła aby serwer je zdeserializował.
